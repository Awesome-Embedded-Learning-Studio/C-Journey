# STM32 入门:从 8051 跨到 32 位 ARM Cortex-M 的第一课

## 引言

笔者大二就开始摸 STM32,但那会儿基本是"照着例程改、能跑就行",葫芦吞枣,始终不扎实。后来回头补 8051 裸机(见 [上一章](./0-8051-bare-metal-basics.md))才想明白一件事:**8051 教会你"寄存器就是硬件的开关",而 STM32 把同一套思想搬到了一个快几十倍、复杂几十倍的 32 位平台上。** 这一章不带你点灯点到最后(那是后续 LED、中断、串口、定时器各章的事),而是先把跨进 STM32 世界的**几个关键认知**讲透:它是什么、芯片型号怎么读、寄存器怎么访问、以及最要命的一个选择——**裸机直操寄存器,还是用 HAL 库**。

> 本章是架构与观念入门,代码片段是寄存器级示例,真正跑起来需要 STM32 板子 + 下载器;和 8051 那章一样,我们先建立"地图",再分章"攻城"。

## STM32 是什么:8051 的"巨炮"表亲

STM32 是 ST(意法半导体)出的一系列 **32 位 ARM Cortex-M 内核**微控制器。拿常见的 `STM32F103ZET6` 举例:

| | 8051(对比) | STM32F103ZET6 |
|---|---|---|
| 内核 | 8 位 8051 | 32 位 ARM **Cortex-M3** |
| 主频 | ~12 MHz(典型) | **72 MHz** |
| Flash | 4–64 KB | **512 KB** |
| SRAM | 128 B–256 B(片内) | **64 KB** |
| 外设 | 少(定时器、串口各一两个) | 8 通用定时器 + 2 高级定时器、3×ADC、5×USART、3×SPI、2×I²C、CAN、USB、SDIO、DMA… |
| 工作电压 | 5 V | **3.3 V** |

一句话:**位宽从 8 变 32、频率快 6 倍、Flash/RAM 大两个数量级、外设多到数不过来。** 但底层模型和 8051 是一脉相承的——都是"CPU 通过读写**寄存器**来控制外设"。

## 看懂型号:STM32F103ZET6 这串字母数字什么意思

ST 的命名规则很规整,看懂了能直接读出任何 STM32 的内核/Flash/封装。拆 `STM32 F 103 ZE T6`:

| 段 | 值 | 含义 |
|---|---|---|
| 家族 | `STM32` | ST 的 32 位 MCU |
| 系列 | `F` | 高性能(F);`L`=低功耗、`G`=经济型、`H`=高性能高安全、`W`=无线 |
| 子型号 | `103` | Cortex-M3 增强型(`101`=基本型、`407`=Cortex-M4…) |
| 引脚数 | `Z` | 144 脚(`C`=48、`R`=64、`V`=100、`Z`=144、`I`=176) |
| Flash 容量 | `E` | 512KB(`C`=256、`E`=512、`I`=2048) |
| 包装/温度 | `T6` | LQFP 封装、工业级温度 |

> 实操:拿到一颗 `STM32F103C8T6`(经典"蓝色药丸"Blue Pill 用的就是它),你能马上读出:Cortex-M3、48 脚、64KB Flash——这就是命名规则的价值,不用查手册先大致心里有数。

## 内存映射寄存器:STM32 操作硬件的核心

8051 的特殊功能寄存器(SFR)在固定地址;STM32 更直接——**所有外设的寄存器都映射到一段统一的内存地址空间**(memory-mapped I/O),你"读写某个地址"就等于"操作某个外设的寄存器"。

C 里怎么访问?——**指向那个地址的指针,且必须 `volatile`**(否则编译器会优化掉对它的读写,因为它不知道这地址背后连着硬件)。这正是 [Chapter 2](../02-pointers-memory/0-pointers-memory-and-bitops.md) 讲 `volatile` 的实战意义:

```c
/* 简化示意:把某个 32 位寄存器地址当成 volatile uint32_t 来读写 */
#include <stdint.h>

#define GPIOA_ODR   (*(volatile uint32_t*)0x4001080C)  /* GPIOA 输出数据寄存器 */

GPIOA_ODR |= (1u << 5);   /* 把 PA5 拉高 → 点亮接在 PA5 的 LED */
```

这里有两层认知,跨过来就通了:

1. **地址即外设**:`0x4001080C` 不是普通内存,是 GPIOA 的输出寄存器;写它就是在改引脚电平。
2. **`volatile` 不可省**:硬件状态随时会变、写出去的命令必须真到硬件,编译器不能把它缓存到寄存器里或合并掉。

实际工程里没人手敲 `0x4001080C`——ST 提供**头文件 + 宏**(如 `GPIOA->ODR`),背后就是上面这套带 `volatile` 的指针解引用。

## 第一道选择题:裸机寄存器 vs HAL 库

这是学 STM32 最容易纠结、也最影响后续路线的选择。三条主流路线:

| 路线 | 怎么写 | 优点 | 代价 |
|---|---|---|---|
| **裸机直操寄存器** | 自己 `RCC->` 开时钟、`GPIOx->` 配模式、写 ODR | 最懂硬件、代码小、最快、可移植性靠自己 | 每个寄存器位都得查手册,入门慢 |
| **HAL 库(ST 主推)** | `HAL_GPIO_WritePin(...)` 等封装函数 | 上手快、CubeMX 图形化生成、跨芯片 | 封装厚、代码臃肿、出问题难调试 |
| **标准外设库(旧)** | 介于两者之间 | 比 HAL 轻量、比裸机省事 | ST 已停更,新芯片不支持 |

> 🔥 **笔者的建议(也是本仓库的取向)**:**先学裸机寄存器,再上 HAL。** 8051 那章你已经在"超级循环 + 直操寄存器"上练过手,STM32 用同样的思路点一个 LED、配一个中断,你会真正明白"HAL 那行 `HAL_GPIO_WritePin` 底层到底干了什么"。直接从 HAL 入门的人,遇到寄存器级 bug 往往两眼一抹黑。本仓库后续的 STM32 各章(LED、按钮、中断、串口、定时器)都**先走寄存器路线**,必要时再对比 HAL。

## 点亮一个 LED:寄存器级的三步骨架

不写具体寄存器值(那是 LED 那章的事),先记住**STM32 操作任何一个外设的通用三步**,这套骨架从 GPIO 到 UART 到定时器全适用:

```c
/* 骨架示意(具体基地址/位定义见参考手册,这里只讲流程) */
void led_init(void) {
    /* ① 开时钟:STM32 的外设默认是断电的,必须先在 RCC 里使能它的时钟 */
    RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;     /* 使能 GPIOA 时钟 */

    /* ② 配模式:把 PA5 配成"推挽输出" */
    GPIOA->CRL &= ~(0xFu << (5 * 4));        /* 先清 PA5 的 4 个配置位 */
    GPIOA->CRL |=  (0x3u << (5 * 4));        /* 设为 输出 50MHz 推挽 */

    /* ③ 操作:写输出数据寄存器,拉高/拉低 PA5 */
    GPIOA->ODR |= (1u << 5);                 /* PA5 = 高,点亮 */
}
```

对比 8051:8051 写 `P1 = 0xFE;` 就点亮 LED,因为 8051 的 IO 上电就能用;STM32 **多了一道"开时钟"**,因为它外设太多,为了省电默认关着。这个"**先开时钟再配寄存器**"是 STM32 新手第一大坑——忘了开时钟,后面全白搭,而且不会有任何报错,只是不工作。

## 常见踩坑

- **忘开外设时钟**:STM32 外设默认不上电。操作 GPIO/UART/定时器前,先在对应 RCC 寄存器里使能时钟,否则寄存器写进去跟没写一样。
- **`volatile` 漏掉**:自己定义寄存器指针不加 `volatile`,开 `-O2` 优化后程序就"不工作"了——因为编译器把对硬件的读写优化没了。
- **3.3V 不是 5V**:STM32 引脚 3.3V,接 5V 外设要电平转换,否则烧片。8051 是 5V 老炮,别混。
- **HAL 上手快但别停在那**:能跑起来 ≠ 懂。每个 HAL 函数都该知道它动了哪个寄存器。

## 小结

从 8051 到 STM32,内核从 8 位变 32 位、外设从几个变几十个,但**底层模型没变:CPU 读写内存映射的寄存器来控制硬件,且这些访问必须是 `volatile`。** 跨过来的关键认知有四条:看懂型号命名、外设操作"先开时钟再配寄存器"的三步骨架、裸机 vs HAL 的路线选择(建议先裸机)、以及 3.3V 电平。把这张地图装进脑子,后面 LED、中断、串口、定时器各章就是按这套骨架逐个外设攻城。
